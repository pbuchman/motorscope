# i18n - Internationalization

This document describes the internationalization (i18n) system used in the Motorscope extension.

## Overview

- **Library**: i18next + react-i18next
- **Primary Language**: English (en)
- **Fallback Language**: English (en)

## File Structure

```
src/i18n/
├── index.ts              # i18n initialization and configuration
└── locales/
    └── en/               # English translations
        ├── common.json   # Common UI text (buttons, labels, statuses)
        ├── dashboard.json # Dashboard-specific text
        ├── popup.json    # Extension popup text
        ├── settings.json # Settings page text
        ├── listing.json  # Car listing details text
        ├── auth.json     # Authentication-related text
        └── errors.json   # Error messages
```

## Namespaces

| Namespace   | Purpose                                          |
|-------------|--------------------------------------------------|
| `common`    | Buttons, generic labels, statuses, vehicle terms |
| `dashboard` | Dashboard titles, filters, empty states          |
| `popup`     | Extension popup UI text                          |
| `settings`  | Settings page labels and descriptions            |
| `listing`   | Detailed car listing information labels          |
| `auth`      | Login/logout, authentication messages            |
| `errors`    | Error messages and confirmations                 |

## Usage in Components

### Basic Usage

```tsx
import { useTranslation } from 'react-i18next';

const MyComponent = () => {
  const { t } = useTranslation('common');
  
  return <button>{t('button.save')}</button>;
};
```

### Multiple Namespaces

```tsx
const MyComponent = () => {
  const { t } = useTranslation(['dashboard', 'common']);
  
  return (
    <div>
      <h1>{t('dashboard:title')}</h1>
      <button>{t('common:button.save')}</button>
    </div>
  );
};
```

### With Interpolation

```tsx
// Translation: "Tracking {{count}} vehicles"
t('dashboard:tracking', { count: 5 })
```

## Key Naming Convention

- Use **dot notation** for hierarchy: `section.subsection.key`
- Use **camelCase** for keys: `button.confirmSave`
- Be **descriptive**: `dashboard.empty.title` vs `emptyTitle`
- Group related keys: `button.save`, `button.cancel`, `button.confirm`

## Adding New Text

1. Add the key and English text to the appropriate namespace file
2. Use `t('namespace:key')` in your component
3. Build and verify the text appears correctly

## Adding a New Language

To add a new language (e.g., Polish):

1. Create new directory: `src/i18n/locales/pl/`
2. Copy all JSON files from `en/` to `pl/`
3. Translate all values (keep keys the same)
4. Update `src/i18n/index.ts`:

```ts
import commonPl from './locales/pl/common.json';
// ... other imports

const resources = {
  en: { /* existing */ },
  pl: {
    common: commonPl,
    // ... other namespaces
  },
};
```

## Best Practices

1. **Never hardcode user-visible strings** in components
2. **Reuse keys** for identical text across the app
3. **Keep translations flat** when possible (avoid deep nesting)
4. **Use interpolation** for dynamic values: `{{name}}`, `{{count}}`
5. **Comment complex keys** in JSON files if needed

## Exceptions

The following strings may remain hardcoded:

- Internal debug/console logs
- Developer-only error messages
- Test descriptions
- Technical identifiers (URLs, API endpoints)
